## Session 2 — Mood Selector + API

**Focus:** Mood strip filter and Supabase data fetching.

### Tasks
- [ ] `MoodSelector` component: horizontal scrollable pill strip
  - Moods: All, Chill, Lively, Date Night, Late Night, Outdoors, Creative
  - Active state styling (coral highlight)
  - Selection updates feed in real-time
- [ ] API route: `GET /api/venues` — query params: `city`, `mood`, `limit`, `offset`
  - Filters by city (required) and mood (optional, maps to vibe tags)
  - Returns venue data with image URLs from Supabase Storage
  - Pagination support
- [ ] Wire discovery feed to API: fetch venues on load + mood change
- [ ] Loading states: skeleton cards while fetching
- [ ] Empty state: "No spots match this mood yet" with illustration
- [ ] Location-based distance calculation (if browser geolocation available, else omit distance)

### Deliverables
- Mood selector filters venues in real-time
- Feed pulls real data from Supabase
- Loading + empty states handled
