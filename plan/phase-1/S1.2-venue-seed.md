## Session 1 — Google Places Ingestion Script ✅

**Focus:** Build the venue seed pipeline from Google Places API.

### Tasks
- [x] Write seed script (`/scripts/seed-venues.ts`):
  - Query Google Places (New) `searchText` API by category × neighborhood
  - Extract: name, address, neighborhood, lat/lng, category, place_id, photos, price level, hours
  - Rate limiting: 55 QPM (safely under 60 QPM limit)
  - Deduplication: skip venues with existing `google_place_id`
  - Batch upsert to Supabase `venues` table
- [x] Define category taxonomy: 10 categories (restaurant, bar, cafe, park, gallery, club, lounge, market, bookstore, rooftop)
- [x] Define neighborhood taxonomy per city:
  - NYC: SoHo, Williamsburg, LES, West Village, Bushwick, East Village, Chelsea, Park Slope, Greenpoint, Harlem, DUMBO, Nolita
  - LA: Silver Lake, Venice, DTLA, Highland Park, Los Feliz, Echo Park, WeHo, Santa Monica, Arts District, Koreatown
  - Chicago: Wicker Park, Logan Square, River North, West Loop, Lincoln Park, Bucktown, Pilsen, Andersonville, Hyde Park, Old Town
  - Miami: Wynwood, South Beach, Little Havana, Brickell, Design District, Coconut Grove, Coral Gables, Midtown, Little Haiti, Edgewater
- [x] npm scripts: `seed`, `seed:nyc`, `seed:dry`
- [ ] Set up Google Places API key + configure billing *(manual step)*
- [ ] Run seed for NYC first as test city *(needs API key)*
- [ ] Verify: `SELECT count(*) FROM venues WHERE city = 'nyc'` → 500+

### Deliverables
- ✅ `scripts/seed-venues.ts` — full seed pipeline with CLI flags
- ✅ `scripts/venue-taxonomy.ts` — category + neighborhood definitions

## Session 2 — Vibe Tags + Image Pipeline ✅

**Focus:** Assign vibe tags to venues, set up image pipeline.

### Tasks
- [x] Define vibe tag vocabulary (21 tags):
  - Energy: chill, lively, rowdy, intimate
  - Setting: rooftop, hidden-gem, outdoor, cozy, spacious
  - Vibe: trendy, classic, artsy, local, upscale, casual, romantic, creative
  - Scene: date-night, group-hang, solo-friendly, late-night
- [x] Auto-assign vibe tags based on:
  - Category → default tags (bar → lively/late-night, cafe → chill/cozy, etc.)
  - Keyword matching on name + description (regex patterns for 22 keyword→tag rules)
  - Price level signals (≥3 → upscale, ≤1 → casual/local)
- [x] Integrated auto-tagging into seed script (venues tagged on ingestion)
- [x] Standalone tag assignment script for re-tagging existing venues
- [x] Image pipeline: Google Places photo URLs stored as `image_url` on venues (800px width)
- [x] npm scripts: `tag`, `tag:nyc`, `tag:dry`
- [ ] Run seed + tag for all cities *(needs API key + Supabase)*
- [ ] Verify: all cities have 500+ venues, each venue has 2-6 vibe tags + image

### Deliverables
- ✅ `scripts/vibe-tags.ts` — 21-tag vocabulary + auto-tagger engine
- ✅ `scripts/assign-vibe-tags.ts` — standalone tag assignment with stats
- ✅ Seed script auto-tags on ingestion
- ⬜ All 4 cities seeded + tagged (needs API keys)
