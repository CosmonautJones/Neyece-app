## Session 2 — Vibe Onboarding Quiz (UI) ✅

**Focus:** Build the 4-screen onboarding quiz with state management.

### Tasks
- [x] Create onboarding route group: `src/app/(onboarding)/quiz/page.tsx`
- [x] Build quiz screen components with:
  - ProgressBar (step X of 5, percentage, animated fill)
  - QuizScreen (title, subtitle, multi-select OptionPills)
  - OptionPill (neo-brutalist toggle with shadow/scale animation)
  - Back + Next navigation with disabled state enforcement
  - Slide animations between screens (slideInLeft / slideInRight keyframes)
- [x] Define quiz data structure in `src/lib/quiz-data.ts`:
  - Screen 1: Going-out energy (chill, lively, rowdy, intimate)
  - Screen 2: Ideal setting (rooftop, hidden gem, outdoor, cozy corner)
  - Screen 3: Vibe words (trendy, classic, artsy, local, upscale, casual) — min 2
  - Screen 4: Deal-breakers (loud, no reservations, far transit, expensive) — min 0
- [x] CitySelect screen after quiz (gated to NYC, LA, Chicago, Miami) with card-style grid
- [x] All answers stored in component state (`QuizAnswers` type) during quiz flow
- [x] Submit wired to `POST /api/onboarding/complete` → redirect to `/discover`

- ✅ Answers captured in typed state
- ✅ City selection with launch city gating
- ✅ Responsive, branded UI with neo-brutalist style + animations
