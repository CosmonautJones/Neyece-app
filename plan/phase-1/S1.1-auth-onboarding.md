# S1.1 — Auth + Onboarding

**Sprint:** Phase 1 · MVP Core
**Sessions:** 3
**Goal:** Clerk auth integrated, vibe onboarding quiz complete, answers stored to DB.

---

## Session 1 — Clerk Auth Integration

**Focus:** Set up Clerk, protect routes, handle sign-in/sign-up flows.

### Tasks
- [ ] Install and configure `@clerk/nextjs`
- [ ] Add Clerk middleware for route protection
- [ ] Create auth pages: `/sign-in`, `/sign-up` (Clerk's pre-built components styled with Neyece brand)
- [ ] Support email + Google sign-in
- [ ] Set up Clerk webhook → create user row in Supabase `users` table on sign-up
- [ ] API route: `POST /api/auth/webhook` — handles Clerk `user.created` event
- [ ] Redirect new users to onboarding, returning users to discovery feed
- [ ] Test: sign up → user appears in both Clerk dashboard and Supabase

### Deliverables
- Auth flow works end-to-end (sign up, sign in, sign out)
- New users automatically created in Supabase
- Routes protected: unauthenticated users redirected to sign-in

---

## Session 2 — Vibe Onboarding Quiz (UI)

**Focus:** Build the 4-screen onboarding quiz with state management.

### Tasks
- [ ] Create onboarding route group: `/app/(onboarding)/quiz/page.tsx`
- [ ] Build quiz screen component with:
  - Progress bar (step X of 4)
  - Question text
  - Multi-select option pills (Neyece-styled)
  - Back + Next navigation
  - Animated transitions between screens
- [ ] Define quiz data structure (4 screens, 3–4 questions each):
  - Screen 1: Going-out energy (chill, lively, rowdy, intimate)
  - Screen 2: Ideal setting (rooftop, hidden gem, outdoor, cozy corner)
  - Screen 3: Vibe words (trendy, classic, artsy, local, upscale, casual)
  - Screen 4: Deal-breakers (loud music, no reservations, far from transit, expensive)
- [ ] City selection screen after quiz (gated to NYC, LA, Chicago, Miami)
- [ ] Store all answers in component state during quiz flow

### Deliverables
- 4-screen quiz navigable with back/next
- Answers captured in state
- City selection works with launch city gating
- Responsive, branded UI matching mockup

---

## Session 3 — Onboarding Backend + Redirect

**Focus:** Save quiz answers to Supabase, generate initial vibe profile, redirect to feed.

### Tasks
- [ ] API route: `POST /api/onboarding/complete` — receives quiz answers + city
- [ ] Write answers to `vibe_profiles` table as JSONB
- [ ] Generate initial vibe fingerprint vector from answers (simple weighted encoding for now — real cosine similarity comes in Phase 2)
- [ ] Update `users` table with selected city
- [ ] Mark user as onboarded (flag in user record)
- [ ] Redirect to discovery feed on completion
- [ ] Handle edge cases: back button, refresh mid-quiz, already-onboarded user
- [ ] Test full flow: sign up → quiz → answers in DB → redirected to feed

### Deliverables
- Quiz answers persisted to Supabase
- Initial vibe profile generated
- Clean redirect flow from auth → quiz → feed
