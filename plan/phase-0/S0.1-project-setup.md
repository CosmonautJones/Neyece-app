## Session 2 — Database & Supabase

**Focus:** Supabase project setup, schema migrations, RLS policies.

### Tasks
- [ ] Create Supabase project (manual step — need project URL + anon key)
- [ ] Write SQL migrations for Phase 0 tables:
  - `users` (id UUID PK, email unique, name, city, created_at, vibe_profile JSONB)
  - `venues` (id UUID PK, name, neighborhood, city, lat, lng, category, google_place_id, vibe_tags JSONB)
  - `vibe_profiles` (user_id FK, answers JSONB, fingerprint_vector, updated_at)
  - `neyece_scores` (venue_id FK, user_id FK, score 0–100, computed_at)
  - `saved_spots` (user_id FK, venue_id FK, saved_at, notes)
  - `waitlist` (email unique, city, created_at, converted boolean)
- [ ] Write RLS policies: users can read/write their own data, venues are public read
- [ ] Create Supabase client utility in `/lib/supabase.ts` (server + client variants)
- [ ] Add database types generation script (`supabase gen types`)
- [ ] Test connection: seed 1 test venue, query it from a server component

### Deliverables
- All 6 tables created with proper indexes and RLS
- Supabase client works in both server and client components
- Type-safe database queries via generated types
