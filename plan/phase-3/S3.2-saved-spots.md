## Session 2 — Collections

**Focus:** User-created lists for organizing saved spots.

### Tasks
- [ ] Create `collections` table: id, user_id, name, description, cover_image, created_at
- [ ] Create `collection_venues` join table: collection_id, venue_id, added_at, notes
- [ ] API routes:
  - `GET /api/collections/[userId]` — list user's collections
  - `POST /api/collections` — create new collection
  - `PATCH /api/collections/[id]` — update name/description
  - `DELETE /api/collections/[id]` — delete collection
  - `POST /api/collections/[id]/venues` — add venue to collection
  - `DELETE /api/collections/[id]/venues/[venueId]` — remove venue
- [ ] Collection presets: suggest "Date Night," "Sunday Morning," "With Friends" on first save
- [ ] Collection detail page: `/app/saved/[collectionId]/page.tsx`
  - Collection name, description, venue count
  - Venue grid (same as saved spots)
  - Edit/delete collection
- [ ] "Add to collection" action when saving a venue:
  - Bottom sheet with collection list + "New Collection" option
- [ ] Share a collection: generates public link with OG image
- [ ] RLS: collections are private by default, public when shared
- [ ] Test: create collection → add 3 venues → share link → verify public view

### Deliverables
- Collections CRUD working
- Add-to-collection flow on venue save
- Shareable collection links with OG image
