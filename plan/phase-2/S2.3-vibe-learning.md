# S2.3 — Vibe Learning Loop

**Sprint:** Phase 2 · Intelligence
**Sessions:** 2
**Goal:** User behavior updates vibe profiles, making the feed more accurate over time.

---

## Session 1 — Signal Tracking

**Focus:** Track implicit and explicit user signals.

### Tasks
- [ ] Create `user_signals` table:
  - id, user_id, venue_id, signal_type, value, created_at
  - Signal types: `save`, `unsave`, `thats_neyece`, `detail_view`, `share`, `time_on_page`
- [ ] Create signal tracking service (`/lib/signals.ts`):
  - `trackSignal(userId, venueId, type, value?)` — inserts to `user_signals`
- [ ] API route: `POST /api/signals/track` — validates and stores signals
- [ ] Wire up signal tracking across the app:
  - Save a venue → `save` signal
  - Unsave → `unsave` signal
  - "That's Neyece" button → `thats_neyece` signal (strong positive)
  - View venue detail → `detail_view` signal
  - Share a venue → `share` signal
  - Time on venue detail page > 10s → `time_on_page` signal
- [ ] Debounce: don't track duplicate signals within 5 minutes
- [ ] RLS: users can only write their own signals, never read others'

### Deliverables
- All user interactions tracked as signals in Supabase
- Signal tracking wired into all relevant UI interactions
- Debouncing prevents signal spam

---

## Session 2 — Profile Update Loop

**Focus:** Periodically update vibe profiles based on accumulated signals.

### Tasks
- [ ] Vibe update function (`/lib/vibe-updater.ts`):
  - Query signals since last profile update
  - Weight signals: `thats_neyece` = 3x, `save` = 2x, `share` = 2x, `detail_view` = 1x
  - For each signaled venue, pull its vibe tags
  - Shift user's vibe vector toward frequently-signaled vibe tags
  - Exponential decay: recent signals weighted more than old ones
  - Normalize vector after update
- [ ] Supabase edge function or cron: run vibe update weekly per user
- [ ] API route: `POST /api/signals/compute-weekly` — trigger for cron job
- [ ] After vibe update → invalidate cached Neyece Scores → trigger score recomputation
- [ ] User-facing: "Your Taste Profile" section on profile page
  - Top 5 vibe tags as colored pills
  - "Your taste is evolving" message if profile changed significantly
- [ ] Re-onboarding prompt: if user inactive > 30 days, offer to retake quiz
- [ ] Test: simulate 20 saves on "chill" venues → verify vibe vector shifts toward chill

### Deliverables
- Vibe profiles update based on real behavior
- Feed gets more accurate over time
- User can see their evolving taste profile
