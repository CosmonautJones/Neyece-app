# S2.2 — AI Insight Generation

**Sprint:** Phase 2 · Intelligence
**Sessions:** 2
**Goal:** Claude API generates personalized "Why this is Neyece for you" insights.

---

## Session 1 — Claude API Integration

**Focus:** Set up Anthropic SDK, build prompt template, generate insights.

### Tasks
- [ ] Install `@anthropic-ai/sdk`
- [ ] Create insight generation service (`/lib/insights.ts`):
  - Input: user vibe profile + venue data (tags, category, neighborhood)
  - System prompt: "You are a knowledgeable local friend who knows this person's taste. In 1–2 sentences, explain why this specific venue matches their vibe. Be warm and specific, never generic."
  - Model: `claude-sonnet-4-6` (fast + cheap for short generations)
  - Max tokens: 100
- [ ] API route: `POST /api/insights/generate` — takes userId + venueId
  - Fetches user profile and venue data
  - Calls Claude API
  - Returns insight text
- [ ] Cache insights in DB: new `insights` table (user_id, venue_id, text, created_at)
  - Check cache before calling API
  - Cache TTL: 7 days (user vibe can shift)
- [ ] Rate limiting: max 10 insight generations per user per minute
- [ ] Fallback: if API fails, return template-based insight (Phase 1 placeholder)
- [ ] Test: generate insights for 5 different user-venue combos, verify quality

### Deliverables
- Claude API generating personalized insights
- Insights cached to avoid repeat API calls
- Fallback working if API is unavailable

### Tasks
