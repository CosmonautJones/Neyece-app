# S2.1 — Neyece Score Engine

**Sprint:** Phase 2 · Intelligence
**Sessions:** 3
**Goal:** Real cosine-similarity scoring between user vibe vectors and venue tag vectors.

---

## Session 1 — Scoring Math Library ✅

### Tasks
- [x] Export `DIMENSIONS` from `fingerprint.ts` for shared use
- [x] `scoring.ts`: pure math functions (no DB calls)
  - `venueTagsToVector()` — maps venue tags to 18-dim space
  - `cosineSimilarity()` — standard cosine similarity
  - `computeVibeMatch()` — cosine sim mapped to 0-100
  - `computeFreshness()` — unsaved = 100, saved = 40
  - `computeMoodAlignment()` — mood tag overlap ratio
  - `computeSocialSignal()` — normalized save + neyece counts
  - `computeNeyeceScore()` — weighted sum (50/20/20/10)
- [x] Sanity test: cosine sim 0.577, score 75 for known vectors

**Design note:** No pgvector needed — 18 dimensions is trivial in TypeScript.
