## Session 2 — Score Computation

**Focus:** Implement the full Neyece Score formula and caching.

### Tasks
- [ ] Implement scoring function (`/lib/scoring.ts`):
  - **Vibe Match (50%):** cosine similarity between user and venue vectors
  - **Freshness (20%):** 1.0 if user hasn't visited/saved, decays over time
  - **Mood Alignment (20%):** current mood tag matches venue primary vibe tag → 1.0, partial → 0.5, no match → 0.0
  - **Social Signal (10%):** normalized count of saves + "That's Neyece" clicks from users with similar vibe profiles
  - Final score: weighted sum × 100, clamped to 0–100
- [ ] Supabase function (or edge function) to compute scores in bulk for a user
- [ ] Cache scores in `neyece_scores` table (user_id, venue_id, score, computed_at)
- [ ] Recompute triggers:
  - On user login (if scores older than 24h)
  - On vibe profile update
  - On mood change (mood component only, lightweight recalc)
- [ ] API route: `GET /api/scores/user/[userId]` — returns cached scores
- [ ] Test: create user with known vibe → compute scores → verify ordering makes sense

### Deliverables
- Full Neyece Score formula implemented
- Scores cached per user-venue pair
- Recomputation triggers working
